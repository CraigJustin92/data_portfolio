Overview:

This project explores the Uber Ride Price Prediction dataset using Python to build a model for estimating fares based on distance, passenger count, and time of day. The workflow includes data cleaning, feature engineering (trip distance, rush hour, seasonality), and log transformation of fares to reduce skewness. Key libraries such as pandas, NumPy, seaborn, and matplotlib were used for data manipulation and visualization, while scikit-learn handled scaling, splitting data, and error metrics (MSE, MAE). The predictive model was built using Ordinary Least Squares (OLS) regression with statsmodels, supported by statistical tests and residual analysis to evaluate fit and assumptions.

Data Preparation

  • Cleaning: Validated coordinates and selected relevant features.
  • Feature Engineering: Calculated trip distance and added a season variable from pickup date.
  • Transformation: Log-transformed fare_amount to reduce skewness.
  • Scaling: Applied StandardScaler for consistent feature scaling.

Key Findings

  • Correlation: Distance strongly correlates with fare (Pearson r = 0.74).
